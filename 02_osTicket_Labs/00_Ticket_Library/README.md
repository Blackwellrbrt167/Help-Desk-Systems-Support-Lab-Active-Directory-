# Help Desk Ticket Library (Practice Queue)

This section simulates a real-world Help Desk ticket queue using osTicket and Active Directory–based infrastructure.

Each ticket is designed to reflect realistic Tier 1–Tier 2 IT support responsibilities, including user management, networking, server maintenance, security incidents, and documentation workflows.

---

##  Ticket Categories

### 01_User_Management_AD
Active Directory–based user support tasks such as onboarding, password resets, account lockouts, and permissions troubleshooting.

### 02_Networking_Infrastructure
Network-related incidents including DNS, DHCP, VPN access, and IP conflicts.

### 03_Server_Workstation_Maintenance
Ongoing maintenance tasks such as patching, disk usage alerts, VM performance issues, and backup failures.

### 04_Security_Incidents
Security-related tickets including phishing investigations, firewall change requests, ransomware simulations, and log audits.

### 05_Documentation_Process
Operational documentation tasks such as knowledge base creation and IT asset inventory audits.

---

##  Ticket Documentation Standard

Each ticket folder contains:
**README.md** – Ticket summary, troubleshooting steps, and resolution
**screenshots/** – Evidence of ticket creation, troubleshooting, and closure
(Optional) **artifacts/** – Logs, commands, or configuration files

All tickets follow this structure:
Problem Statement
Business Impact
Root Cause
Resolution Steps
Preventive Recommendation

---

##  Purpose

This ticket library demonstrates:
Practical Help Desk workflows
